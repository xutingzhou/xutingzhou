- Kotlin声明的类默认是 `final` 和 `public`。嵌套的类默认不是内部类。
- `override` 修饰符用来标注被重写的父类或者接口的方法和属性，使用 `override` 修饰符是强制要求的。
- 接口的方法可以有一个默认实现，不需要特殊的注解。
- 一个类实现的两个接口中的方法签名相同，并都有默认实现方法，Kotlin编译器会强制要求你提供自己的实现。
- 允许创建一个类的子类，使用 `open` 修饰符标记这个类。
- 抽象类的抽象成员始终是 `open` 的，不需要显示使用 `open` 修饰符。非抽象函数默认不是 `open`，可以标注为 `open`。
- 接口中的成员始终是 open 的，不能声明为 final。如果没有函数体，就是 abstract，但是这个关键字不是必须的。
- |修饰符|相关成员|评注|
  |final（默认）|不能被重写|类中成员默认使用|
  |open|可以被重写|需要明确表明|
  |abstract|必须被重写|只能在抽象类中使用；抽象成员不能有实现|
  |override|重写父类或接口中的成员|如果没有使用final表明，重写的成员默认是open|
- 可见性修饰符：默认为public
	- Kotlin只把包作为在命名空间里组织代码的一种方式使用，并没有将其用作可见性控制。
	- |修饰符|类成员|顶层声明|
	  |public（默认）|所有地方可见|所有地方可见|
	  |internal|模块中可见|模块中可见|
	  |protected|子类中可见|-|
	  |private|类中可见|文件中可见|
	- 类的基础类型和类型参数列表中用到的所有类，或者函数的签名都有与这个类或者函数本身相同的可见性。
- 嵌套类与内部类：默认嵌套类
	- Kotlin的嵌套类不能访问外部类的实例。
	- Kotlin中没有显示修饰符的嵌套类与Java中的 `static` 嵌套类是一样的。
	- 要把嵌套类变成一个内部类来持有一个外部类的引用的话需要使用 `inner` 修饰符。使用 `this@Outer` 从 Inner 类去访问 Outer 类。
- 密封类
	- 为父类添加 `sealed` 修饰符，对可能创建的子类做出严格的限制。
	- 所有的直接子类必须嵌套在父类中。
	- `sealed` 修饰符隐含的这个类是一个 `open` 类。
- 接口
	- 接口没有构造函数
	- 接口中可以包含 `getter` 和 `setter` 的属性，只要没有引用支持字段。
- 数据类

	- 有通用对象方法：`equals`, `hashCode`, `toString`

	- 使用copy来创建或修改。创建副本是修改实例的好选择，副本有独立的生命周期并且不会影响代码中引用原始实例的位置。

- `lambda` 表达式，可以传递给其他函数的一小段代码。
- 作为函数参数的代码块
- 如果 `lambda` 刚好是函数或者属性的委托，可以用成员应用替换
- Kotlin的 `lambda` 表达式始终用花括号包围
- 如果 `lambda` 表达式是函数调用的最后一个实参，可以放到括号的外边
- 当 `lambda` 是函数唯一的实参时，可以去掉调用代码中的空括号对
- ```Kotlin
  people.maxBy({ p: Person -> p.age })
  people.maxBy(){ p: Person -> p.age }
  people.maxBy{ p: Person -> p.age }
  ```
- 从上下文中捕捉变量
	- Kotlin中不仅限于访问 `val` 变量，可以在 `lambda` 内部访问和修改 `var` 变量
	- 变量被 `lambda` 捕捉
		- 捕捉了一个 `val` 变量，它的值被拷贝下来
		- 捕捉了一个 `var` 变量，它的值被作为 `Ref` 类的一个实例被存储下来
	- 如果 `lambda` 被作用事件处理器或者用在其他异步执行的情况，对局部变量的修改只会在 lambda 执行的时候发生

		- 对同一个对象执行多次操作，不需要反复把对象的名称写出来

	- `apply`

- 内联函数
	- 消除 `lambda` 带来的运行时开销（每次调用就会创建一个新的对象）
	- 使用 `inline` 修饰符标记一个函数，在函数被使用时，编译器不会生成函数调用的代码，而是使用函数实现的真实代码替换每次的函数调用。
	- 不是所有使用 `lambda` 的函数都可以被内联。一般来说，参数如果被直接调用或者作为参数传递给另一个 `inline` 函数，是可以内联的。
	- 接收非内联 `lambda` 的参数，用 `noinline` 修饰符标记。
- use 函数是一个扩展函数，被用来操作可关闭的资源，接收一个 lambda 作为参数。
- lambda 中的返回语句
	- 在 lambda 中使用 return 关键字，会从调用 lambda 的函数中返回，并不是从 lambda 中返回。这样的 return 语句叫非局部返回。
	- 只有在以 lambda 作为参数的函数是内联函数的时候，才能从更外层的函数返回。
	- lambda 中的局部返回，终止 lambda 的执行，并接着从调用 lambda 的代码处执行。要用标签。
	- 要标记一个 lambda 表达式，在 lambda 的花括号之前放一个标签名，接着放一个 @ 符号。要从一个 lambda 返回，在 return 关键字后放一个 @ 符号，接着放标签名。
		- ```Kotlin
		  people.forEach label@{
		    if(it.name == "A") return@label
		  }
		  ```
	- 使用 lambda 作为参数的函数的函数名作为标签
		- ```Kotlin
		  people.forEach{
		    if(it.name == "A") return@forEach
		  }
		  ```
	- 一个 lambda  表达式的标签数量不能多于一个
- `lambda` 的局部返回语法相当冗长，可以使用匿名函数来传递代码块
- 在匿名函数中，不带标签的 `return` 表达式会从匿名函数返回，而不是从包含匿名函数的函数返回。规则：`return` 从最近的使用 `fun` 关键字声明的函数返回。
- 匿名函数是 `lambda` 表达式的另一个语法形式

- 有时候会把类型和类当成同样的概念时候，但它们不一样。
- `子类型`：任何时候如果需要的是类型A的值，你都能使用类型B的值来当作A的值，类型B就称为类型A的子类型。
- `超类型`：子类型的反义词。如果A是B的子类型，B就是A的超类型。
- 一个泛型类，例如MutableList，如果对于任意两种类型A和B，MutableList<A>既不是MutableList<B>的子类型也不是它的超类型，它就被称为在该类型参数上是`不变型`的。
- Java中所有的类都是不变型的，虽然那些类具体的使用可以标记成可变型的。
- `协变`
	- 保留子类型化关系
	- 如果A是 B 的子类型，那么List<A>就是List<B>的子类型，就是子类型化被保留了，这样的类或者接口被称为`协变`的
	- `out`关键字声明类在某个类型参数上是可以协变的
	- 将类型参数标记为协变的，在该类型实参没有精确匹配到函数中定义的类型形参时，可以让该类的值作为实参传递，也可以作为返回值
	- 不能把任何类都变成协变的，这样不安全
	- `out`位置，函数把T当成返回值，该函数`生产`类型为T的值
	- `in`位置，T用作函数参数的类型，该函数`消费`类型为T的值
	- 如果在类的构造方法的参数上使用了val和var，就会同时声明一个getter和一个setter（如果属性是可变的）。对于只读属性来说，类型参数用在了out位置，而可变属性在out位置和in位置都使用了。
	- 位置规则值覆盖类外部可见的API（public，protected和internal），私有方法的参数既不在in位置也不在out位置。
	- 变型规则只会防止外部使用者对类的误用，但不会对类自己的实现起作用。
- `逆变`
	- 反转子类型化关系
	- 逆变的概念可以被看成是协变的镜像
	- 如果B是A的子类型，T<A>是T<B>的子类型，类型参数A和B交换了位置，就说子类型化被反转了。
- |协变|逆变|不变型|
  |Producer<out T>|Consumer<in T>|MutableList<T>|
  |类的子类型化保留|子类型化反转了|没有子类型化|
  |T只能在out位置|T只能在in位置|T可以在任何位置|
- 一个类可以在一个类型参数上协变，同时在另外一个类型参数上逆变。
- `使用点变型`
	-
