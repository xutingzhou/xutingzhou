- 几种使程序更安全的技巧
	- 避免可变引用（变量）并在无法避免突变时抽象单例
	- 避免控制结构
	- 将作用（与外界的交互）限制在代码中的特定区域
	- 没有异常抛出
- 副作用，当方法或函数返回一个值并产生作用时
- 既不会改变也不依赖于外部世界的代码被认为是引用透明的
- 引用透明的代码的属性
	- 独立性
	- 确定性
	- 永远不会抛出任何异常
	- 不会创建导致其他代码意外失败的条件
	- 不依赖于任何外部设备来工作
- 安全编程的好处
	- 程序将更容易推理分析
	- 程序将更容易测试
	- 程序将更模块化
	- 程序的组合和重组更容易
	- 本质上是线程安全的
-
- 惰性初始化
	- 使用by lazy{}初始化
	- 只能用于val声明的不可变变量
- 延迟初始化
	- 使用lateinit初始化
	- 只能用于var声明的可变变量
- 一个好的编程习惯是只允许对专门为它设计的类进行扩展
- 在Kotlin中，类没有静态成员。要获取相同的效果，必须使用伴生对象的特殊结构
  ```Kotlin
  data class Person(val name: String) {
    companion object {
      fun create(xml: String){
        ....
      }
    }
  }
  ```
- 单例模式，对象不能有构造函数，如果有属性，必须初始化或抽象
- Kotlin没有原语
	- 原语，primitive，也称基元，通常表示基本数据类型
- lambda
	- 多行lambda表达式返回的值是最后一行表达式的值，可以使用return关键字，也可以省略
	- 推荐做法是吧lambda表达式写成几行
- 使用表达式和函数替换控制结构
- 型变描述了参数化类型与子类型之间的关系
